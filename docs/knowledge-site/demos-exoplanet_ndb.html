exoplanet_ndb

Confirmed exoplanets from the NASA Exoplanet Archive, browsed as N-ary discovery facts. Every dot in the scatter is one PLANET entity. Every discovery is one arity-4 hyperedge [star, planet, method, mission]. Every multi-planet system is one arity-(N+1) hyperedge — Kepler-90 rolls up as arity 9, TRAPPIST-1 as arity 8.

Launch live demo → NASA Exoplanet Archive

⚠ The 91-planet curated seed is a historically-chosen subset of the ~6,300 confirmed planets in the archive (May 2026). The live-fetch form lets you grow it to any planet in the public table.

What this is

A single-page web app that lets you walk through the history of exoplanet hunting the way nDB stores it:

  • 91 planets across 40 systems, picked to cover all 11 NASA-recognised detection methods (transit, radial velocity, microlensing, direct imaging, astrometry, pulsar timing, TTV, ETV, OBM, pulsation-timing, disk kinematics) and the historical timeline 1992–2025.
  • One arity-4 DISCOVERY hyperedge per planet. A single record holds star + planet + method + mission, with year / facility / telescope as edge properties. The same fact in SQL is a JOIN across discovery_method + discovery_mission tables.
  • One arity-(N+1) SYSTEM hyperedge per system. TRAPPIST-1 = 1 star + 7 planets = arity 8. Kepler-90 = 1 star + 8 planets = arity 9 (the demo's tallest).
  • One HABITABLE_ZONE hyperedge per system that has HZ planets (TOI-700, TRAPPIST-1, Kepler-186, Proxima Cen, …).
  • Live NASA TAP fetch. Type any planet name in the sidebar — one row pulled from the archive, committed to nDB as one PLANET entity + one DISCOVERY hyperedge.

The 30-second tour

  1. Look at the M-R scatter. X axis = planet radius (Earth radii, log). Y axis = planet mass (Earth masses, log). Every dot is one PLANET entity. Color = discovery method. The yellow-bordered dots are habitable-zone candidates.
  2. Click a method in the left legend. It mutes that method's dots — useful for "what did Kepler see vs what Hubble imaged?"
  3. Click any dot. The right pane fills with the planet's record, the host star's record, the arity-4 DISCOVERY hyperedge with its four role-fillers, and the arity-(N+1) SYSTEM hyperedge listing every co-planet.
  4. The orbital diagram appears at the top of the right pane — co-planets rendered as concentric scaled orbits with animated dots. The selected planet is the highlighted ring; HZ orbits are dashed yellow.
  5. Switch the centre to "Discovery year" in the top-right toggle — a stacked-bar histogram of who detected what when. Pulsar timing peaks in 1992; Kepler's wave is 2014–2016; TESS's is 2020–.
  6. Open the "What does nDB store?" disclosure at the bottom of the right pane to see the raw wire-format JSON for the PLANET entity and its DISCOVERY hyperedge.

Why "N-dimensional database" for this dataset

Most exoplanet databases are giant tables. The NASA Exoplanet Archive's "Planetary Systems" view, for example, is one row per planet with hundreds of columns. That works for filtering and aggregation — but it loses the fact that several distinct kinds of relationships live on top of each planet:

FactTrue aritySQL equivalent
Planet discovered by a method using a mission4Joins to method + mission + year + team tables
Multi-planet system (TRAPPIST-1)81 row in system + 7 rows in system_member
Multi-planet system (Kepler-90)91 row in system + 8 rows in system_member
HR 8799 direct-imaged system5Same junction pattern
Habitable-zone group (TOI-700)3Filter query + materialised view

In nDB you don't reify. The relationship is the record. A SYSTEM hyperedge has type 101, arity 9, and role-fillers [star=Kepler-90, planet=b, c, i, d, e, f, g, h] — that's the whole story. Click the system in the dropdown and the orbital diagram is reading from one record.

Schema

Kept in lockstep between crates/ndb-renderer/examples/exoplanet_explorer/main.rs and docs/exoplanet/index.html.

Entity types

  • T_STAR = 1 — one entity per unique host star (40 in the seed)
  • T_PLANET = 2 — one entity per planet (91)
  • T_METHOD = 3 — one entity per detection method (11)
  • T_MISSION = 4 — one entity per facility / mission (21)

Hyperedge types

  • T_DISCOVERY = 100 — arity 4: [STAR, PLANET, METHOD, MISSION], with year/facility/telescope as edge properties
  • T_SYSTEM = 101 — arity 2..9: [STAR, PLANET, PLANET, …]
  • T_HABITABLE_ZONE = 102 — arity 2..M+1: [STAR, HZ_PLANET, …]

Data provenance

Every number in this demo comes from the NASA Exoplanet Archive's "Planetary Systems" table (default_flag=1 rows) — the same machine-readable source used by published exoplanet review papers. The curated 91-planet picks file is at tools/exoplanet/build_seed.py and re-running it after fetching a fresh CSV regenerates seed.json deterministically.

  • 1992 pulsar planets — Wolszczan A. & Frail D. 1992. Nature 355:145.
  • 51 Peg b (1995) — Mayor M. & Queloz D. 1995. Nature 378:355. (Nobel Prize in Physics 2019.)
  • HD 209458 b transit (1999) — Charbonneau D. et al. 2000. ApJ 529:L45.
  • OGLE-2005-BLG-390L b microlensing — Beaulieu J.P. et al. 2006. Nature 439:437.
  • 2M1207 b direct imaging (2004) — Chauvin G. et al. 2005. A&A 438:L25.
  • HR 8799 system — Marois C. et al. 2008, 2010. Science 322:1348; Nature 468:1080.
  • TRAPPIST-1 seven planets — Gillon M. et al. 2017. Nature 542:456.
  • Kepler-90 eighth planet — Shallue C.J. & Vanderburg A. 2018. AJ 155:94.
  • HD 110067 resonant chain — Luque R. et al. 2023. Nature 623:932.

What this is not

  • Not a replacement for the NASA Exoplanet Archive. Treat the local nDB as a curated working set. The archive remains the source of record.
  • Not full-archive. 91 of ~6,300 confirmed planets, picked to tell a story across all detection methods. The fetch form lets you grow it; the engine itself scales to millions.
  • Not multi-default-handling. The archive has many rows per planet (one per published refinement). The seed uses only the row with default_flag=1.
  • Not 3D galactic position — RA/Dec are stored but the 3D sky map is deferred to v2.

Run it yourself

git clone https://github.com/goldrag1/nDB
cd nDB
cargo run -p ndb-renderer --example exoplanet_explorer
# then open http://127.0.0.1:9877/

The engine state lives at <workspace>/.demo-data/exoplanet-ndb (project-local, gitignored). Planets you add via the fetch form persist between runs.
