Live race — nDB vs Postgres

Both sides loaded the same 100,000-record shape on startup: 50,000 entities (49k customers + 1k regions) + 50,000 hyperedges (45k sales junctions + 5k region-containment edges). The schemas are honest — Postgres uses the natural junction-table encoding with indexes on every join key and ANALYZE run before this page went live. Same Linux host, same kernel, no Docker, single-threaded.

Pick a workload, hit Race. Both backends run the workload N times in parallel and report wall-clock μs at the user-call boundary (p50, p99, throughput). The winner's card lights up. Numbers vary ~10-30% run-to-run — the gap is the signal, not the headline number. Rate-limited to one race per workload per 3 seconds.
