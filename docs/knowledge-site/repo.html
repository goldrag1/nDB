What ships in v2.3

  • Engine format v2 — u32 arity (max 4.3B; v1 was u8, capped at 255 — KRAS at 1517 atoms broke that)
  • Atoms as first-class nDB entities (type 7) with property codes 60–69 (NAME, ELEMENT, SERIAL, X/Y/Z, BFACTOR, CHAIN, RES_POS, RES_NAME)
  • N-ary protein_atoms hyperedge (arity 1518 for KRAS) — replaces 1517 binary atom_of edges in v2.2
  • Persistent DB survives restarts; seeds only when DB has no proteins
  • Engine format breaking change accepted — v1 DBs cannot be opened by v2 code

Install + run

# Clone
git clone https://github.com/goldrag1/nDB
cd nDB

# Build + test everything
cargo build --workspace
cargo test --workspace

# Launch the alphafold demo
cargo run -p ndb-renderer --example v22_explorer
# Opens two ports: 8742 (API), 9876 (web app)
# Browse to http://127.0.0.1:9876/

# Stop with Ctrl-C

Demo state persists at <workspace>/.demo-data/alphafold-ndb. To start fresh, delete that directory.

Roadmap

  • v3.0 — Analytics + first GPU support. Columnar index plugin (CPU + CUDA), GPU vector similarity, dataset slicer.
  • v4.0 — Distribution + inference. Multi-writer, distributed transactions, MCP server with inference rules.
  • v4.x — High-dimensional embeddings as first-class. Vector index + structured pattern in one query plan.

Detailed milestones in the repo README.

Contributing

Issues, PRs, and "I tried this and it broke" emails are welcome. Style: small commits, descriptive messages, tests cover the change. The Rust workspace is conventional: cargo fmt, cargo clippy --all-targets --all-features -- -D warnings, cargo test --workspace all clean before opening a PR.
